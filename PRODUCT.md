# SplitEase 產品說明書

## 1. 產品概述

- **產品名稱**：SplitEase
- **產品類型**：手機優先（Mobile-first）分帳 Web App
- **核心功能**：簡單快速的分帳工具，適用於晚餐聚會、日常消費分攤
- **資料儲存**：瀏覽器本地端（localStorage），無需登入

---

## 2. 使用情境

### AA制（大家平分）
- 一人先出錢
- 其他人平均分攤
- 可選擇指定付款人

### AB制（各付各的）
- 每個人輸入自己實際消費的金額
- 共享成本（如外送費）自動平均分攤

---

## 3. 功能列表

### 3.1 專案管理
| 功能 | 說明 |
|------|------|
| 建立專案 | 輸入名稱 + 選擇模式（AA/AB）+ 加入成員 |
| 刪除專案 | 確認後刪除，資料不可復原 |
| 專案列表 | 顯示所有專案，標示模式與總金額 |

### 3.2 成員管理
| 功能 | 說明 |
|------|------|
| 新增成員 | 輸入名字加入 |
| 移除成員 | 從專案中移除 |

### 3.3 消費紀錄
| 功能 | 說明 |
|------|------|
| 新增消費 | AA制：輸入金額，選擇誰付<br>AB制：選擇誰吃了，輸入金額 |
| 消費列表 | 顯示所有消費紀錄與日期 |
| 自動計算 | 自動計算每人應付金額 |

### 3.4 結算
| 功能 | 說明 |
|------|------|
| 查看結欠 | 顯示每人應付/應收金額 |
| 一鍵結清 | 清除所有消費紀錄，專案保留 |
| 刪除專案 | 連同消費紀錄一起刪除 |

---

## 4. 操作流程

### 建立專案
```
1. 輸入專案名稱
2. 選擇 AA 或 AB 模式
3. 加入成員（至少一人）
4. 點擊建立
```

### 新增消費（AA制）
```
1. 輸入消費項目
2. 輸入金額
3. 選擇誰付錢（可選）
4. 儲存
```

### 新增消費（AB制）
```
1. 選擇誰吃了
2. 輸入金額
3. 儲存
```

### 查看結欠
```
1. 進入專案
2. 查看結欠頁面
3. 顯示每人應付金額
```

## 5. UI/UX 設計

### 設計原則
- 手機優先，單手可操作
- 大按鈕，適合拇指觸控
- 底部浮動主要按鈕
- 視覺化區分 AA（紅色）/ AB（綠色）

### 配色
| 用途 | 顏色 |
|------|------|
| 主色 | #6366f1（紫） |
| AA制 | #ef4444（紅） |
| AB制 | #22c55e（綠） |
| 背景 | #f8fafc |

---

## 6. 技術規格

- **前端**：單一 HTML 檔，無框架依賴
- **儲存**：localStorage
- **部署**：Vercel（靜態部署）
- **相容性**：現代瀏覽器，手機優先

---

## 7. 限制

- 不支援登入
- 不支援分享（需自行記錄邀請碼）
- 不支援跨裝置同步
- 清除瀏覽器資料會遺失
